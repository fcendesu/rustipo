# Rustipo v0.4.0 Checklist

Historical scope tracker for the original v0.4.0 planning batch.
Current status is preserved here for traceability.

## Scope lock

- [x] Favicon support
- [x] Mermaid support
- [x] Generic nested custom pages
- [x] Theme variants as explicit theme IDs

## Execution order

### 1) Favicon support (completed)

- [x] Support `favicon.ico` and `favicon.svg` from `static/`
- [x] Add optional config path (default fallback to `/favicon.ico`)
- [x] Inject favicon links in base template context
- [x] Fail with readable error if configured favicon is missing
- [x] Add tests for asset copy + template output

### 2) Mermaid support

- [x] Detect fenced code blocks with `mermaid`
- [x] Render Mermaid container HTML for those blocks
- [x] Keep normal syntax highlighting for non-Mermaid code fences
- [x] Inject Mermaid runtime only on pages that use Mermaid
- [x] Add tests for render behavior and runtime injection gating

### 3) Generic nested custom pages

- [x] Support nested routes outside `blog/` and `projects/`
- [x] Keep special handling for `blog/` and `projects/` sections
- [x] Preserve slug normalization and frontmatter slug override
- [x] Add tests for nested route mapping and invalid path handling

### 4) Theme variants (explicit IDs)

- [x] Define variant naming convention (`family-variant`)
- [x] Prototype variant naming for selectable IDs
- [x] Ensure `theme list` prints variant IDs clearly
- [x] Document recommended variant usage in config

Later product direction moved color presets into a separate palette system.

## Out of scope for v0.4.0

- Plugin system
- Theme registry
- Full shortcode/plugin API redesign
- Search ranking improvements
- i18n

## Release gate

Original gate for this batch:

- [x] All four scope items above are complete
- [x] `cargo fmt`, `cargo clippy --all-targets --all-features -- -D warnings`, and `cargo test` pass
- [x] Docs are updated for shipped favicon behavior
- [x] Historical note added that color presets moved into the palette system
